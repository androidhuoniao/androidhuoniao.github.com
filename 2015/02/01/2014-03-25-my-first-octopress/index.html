如何搭建一个octopress博客

为什么要搭建octopress博客？

最近半年，觉着自己没有什么进步，虽然每天做了很多的工作，但是几乎没有什么积累，究其原因，在于没有记录，没有总结，遂萌发了搭建一个自己博客的念头，看了网友一个朋友的文章“我为什么要从wordpress转到octopress，行了，就是它了，开始搭建了。

遇到的问题

网络上的octopress搭建教程多如牛毛，貌似都很简单，但是我却遇到了很多的问题，总结出一个规律，不看什么教程，直接去octopress看引导是最保险的。

先来记录下一些基本的知识点

  • Ruby：octopress框架的实现语言。rvm（ruby version manager）是用于管理ruby版本的，rake是ruby中类似于make工具。
  • 静态站点生成工具：简单地说，不用数据库，直接生成网页文件。Jekyll就是这样的工具，而octopress是构建于它的上层的框架。
  • Git：我主观的认为完虐svn的分布式版本控制工具。
  • Github：程序员的Facebook,为程序员托管了很多代码的站点。
  • Github pages:github推出的，给与程序员自由创造静态网页的功能。支持Jekyll，因此也支持octopress。
  • homebrew：mac os下的软件包管理工具，类似于linux下的dpkg。它使用ruby脚本，mac os下自带了ruby。
  • octopress 其官网的help中有搭建octopress的足够的权威指导
  • YAML: YAML 试图用一种比 XML 更敏捷的方式，来完成 XML 所完成的任务。在octopress中，是使用yaml做配置文件的书写的

依赖库

  • homebrew 之前一直使用macport，homebrew比macport更为方便
  • gcc mac中去掉了，需要使用homebrew独自安装
  • command line 也需要独立安装 xcode-select —install
    这些都必须在安装octopress之前安装好
安装Ruby

有两个Ruby管理工具，Rvm和Rbenv，两个都行，octopress上安装指南
“Rvm install guide”
“Rbenv install guide”
